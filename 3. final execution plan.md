## A.1 The Problem We Are Solving

**Problem Statement:**

Before making any investment decision — buying stock, funding a startup, acquiring a company, or advising a client — professionals and individuals need to deeply understand a company across multiple dimensions: financial health, legal standing, governance quality, and market perception.

**The current state of the world:**

| How it's done today | Cost | Time | Who can afford it |
|---|---|---|---|
| Big 4 consulting firms (Deloitte, KPMG, PwC, EY) | ₹10–50 lakhs per report | 3–8 weeks | Only PE/VC firms and corporates |
| Boutique research firms (Tofler, VCCEdge) | ₹500–5,000/report or ₹50K+/month | 1–3 days (stale database) | Mid-size firms |
| Bloomberg Terminal | $24,000/year | Real-time but requires expertise | Institutional only |
| Manual DIY research | Free (costs time) | 20–40 hours across 10–15 websites | Anyone with patience |

**The 3 core pain points:**

1. **Cost Barrier** — 10 Cr+ retail investors in India and thousands of small CAs/advisors are completely locked out of professional-grade company research
2. **Time Barrier** — Even if you know what to look for, visiting Screener.in, BSE, NSE, Google News, and MCA portal manually takes 20–40 hours of tedious copy-paste work
3. **Data Fragmentation** — Critical data is scattered across 10+ websites, all with different interfaces, some behind logins, none with public APIs. No single product aggregates live data from all of them

**Why this is a perfect TinyFish problem:**

Every single data source our product needs requires **real browser automation** — not API calls, not database lookups, not web scraping libraries:

| Data Source | Why it needs TinyFish (real browser) |
|---|---|
| Screener.in | JavaScript-rendered financial tables, dynamic tab navigation, lazy-loaded quarterly data |
| BSE India (bseindia.com) | Corporate announcements require search → filter → paginate. Shareholding pattern requires dynamic form submission |
| NSE India (nseindia.com) | Heavy anti-bot protection, JavaScript-rendered corporate filings, requires session cookies |
| Google News | Dynamic search results page, infinite scroll, cards with nested article data |
| Economic Times | Topic pages with dynamically loaded article cards, JS rendering |
| MCA Portal (post-MVP) | Requires login, CIN search, multi-page navigation, CAPTCHA handling |
| Glassdoor (post-MVP) | Requires login, anti-scrape walls, dynamically rendered review data |
| LinkedIn (post-MVP) | Requires login, aggressive bot detection, JS-rendered company pages |
| Moneycontrol (post-MVP) | Login for analyst section, dynamically rendered recommendation tables |

**The test from TinyFish's hackathon page:**
> *"If your application can be built without a web agent navigating real websites, it is not a fit."*

Our application CANNOT be built without TinyFish. Every data point comes from navigating live websites in a real browser.

## A.2 How We Are Solving It

**Solution: CompanyIQ — the Credit Score for Companies**

An AI-powered Company Due Diligence Agent that:
1. Takes a company name as input
2. Deploys **3 specialized browser agents in parallel** using the TinyFish Web Agent API
3. Each agent navigates real websites (Screener.in, BSE India, Google News, Economic Times) to extract live data
4. A **computation engine** processes raw data into 25+ financial ratios, legal risk scores, and sentiment scores
5. A **red flag detection engine** automatically identifies critical risks
6. An **LLM synthesis layer** (Claude API) generates a human-readable executive summary and investment thesis
7. Delivers a complete **CompanyIQ Report** with a branded 0–100 score in under 3 minutes

**How TinyFish is used (this is the core of the submission):**

```
USER types "Zomato"
    │
    ├─► FINANCIAL AGENT ──► TinyFish opens browser ──► Screener.in
    │     3 parallel calls     navigates to P&L tab     extracts 8 quarters of data
    │                          navigates to Balance Sheet  extracts assets, liabilities
    │                          navigates to Cash Flow     extracts 4 years of flows
    │
    ├─► LEGAL AGENT ──► TinyFish opens browser ──► BSE India
    │     3 parallel calls    searches company name     extracts 20 announcements
    │                         navigates shareholding    extracts pledging, FII/DII data
    │                         extracts director list    identifies resignations
    │
    └─► SENTIMENT AGENT ──► TinyFish opens browser ──► Google News + ET
          2 parallel calls      searches company news     extracts 25 headlines + snippets
                                navigates topic page      extracts article summaries
    │
    ALL 8 TinyFish calls run in PARALLEL (3 concurrent agents)
    │
    ▼
    Computation Engine → 25 ratios + 3 pillar scores + red flags
    ▼
    LLM Synthesis → Executive summary + investment thesis
    ▼
    CompanyIQ Report delivered to user
```

**Total per report: 8 TinyFish API calls, ~90–132 browser steps, ~2–3 minutes**

## C.3 TinyFish API Integration Layer

The most important module in the entire project. Every agent depends on this.

**`backend/agents/tinyfish.js` — Specification:**

```javascript
/**
 * callTinyFish(url, goal, options) → Promise<{resultJson, steps, duration}>
 *
 * @param {string} url        — Target website URL
 * @param {string} goal       — Plain English instruction
 * @param {object} options    — Optional config
 * @param {string} options.browserProfile  — "lite" or "stealth" (default: "lite")
 * @param {object} options.proxyConfig     — { enabled: bool, country_code: string }
 * @param {number} options.timeout         — Max wait in ms (default: 90000)
 * @param {number} options.retries         — Retry count on failure (default: 1)
 * @param {function} options.onProgress    — Callback for PROGRESS events
 *
 * @returns {object} { resultJson, stepCount, durationMs, runId }
 *
 * Behavior:
 * 1. POST to https://agent.tinyfish.ai/v1/automation/run-sse
 * 2. Parse SSE stream: STARTED → PROGRESS (n times) → COMPLETE
 * 3. On each PROGRESS event, call onProgress(purpose) if provided
 * 4. On COMPLETE, return resultJson
 * 5. On timeout or error, retry once with same params
 * 6. On final failure, return { error: message, resultJson: null }
 */
```

**Key implementation details:**
- Use `node-fetch` with streaming response body
- Parse SSE format: lines starting with `data: ` contain JSON payloads
- Count PROGRESS events to track step count
- Measure wall-clock time from STARTED to COMPLETE
- The `onProgress` callback powers the real-time UI loading experience

```javascript
const BENCHMARKS = {
  netProfitMargin:  { poor: 0, avg: 5, good: 12, excellent: 20 },
  roe:              { poor: 0, avg: 8, good: 15, excellent: 22 },
  ebitdaMargin:     { poor: 5, avg: 15, good: 25, excellent: 35 },
  currentRatio:     { danger: 0.8, poor: 1.0, avg: 1.5, good: 2.0, excellent: 3.0 },
  debtToEquity:     { excellent: 0, good: 0.5, avg: 1.0, poor: 2.0, danger: 3.5 },
  revenueCAGR3yr:   { poor: 0, avg: 8, good: 15, excellent: 25 },
  pe:               { // Context-dependent: compare to sector avg }
};

function scoreRatio(value, benchmark, isInverse = false) {
  // Maps a ratio value to 0-100 based on benchmark thresholds
  // isInverse = true for ratios where lower is better (debt/equity, P/E)
  if (value == null) return 50; // Neutral score for missing data
  
  if (!isInverse) {
    if (value >= benchmark.excellent) return 95;
    if (value >= benchmark.good) return 75 + (value - benchmark.good) / (benchmark.excellent - benchmark.good) * 20;
    if (value >= benchmark.avg) return 50 + (value - benchmark.avg) / (benchmark.good - benchmark.avg) * 25;
    if (value >= benchmark.poor) return 25 + (value - benchmark.poor) / (benchmark.avg - benchmark.poor) * 25;
    return Math.max(0, 25 * (value / Math.max(benchmark.poor, 0.01)));
  } else {
    // For inverse ratios (lower = better, e.g., Debt/Equity)
    if (value <= benchmark.excellent) return 95;
    if (value <= benchmark.good) return 75;
    if (value <= benchmark.avg) return 50;
    if (value <= benchmark.poor) return 25;
    return Math.max(0, 10);
  }
}
```

```javascript
  } else if (directorChanges >= 2) {

    flags.push({ severity: "WATCH", message: `${directorChanges} director changes in 12 months` });

  const pledge = legalData.shareholding?.latest?.pledgedPercent || 0;
  
  if (pledge > 75) {
    score -= 35;
    flags.push({ severity: "CRITICAL", message: `Promoter pledging at ${pledge}% — extreme risk` });
  } else if (pledge > 50) {
    score -= 25;
    flags.push({ severity: "HIGH", message: `Promoter pledging at ${pledge}% — high risk` });
  } else if (pledge > 30) {
    score -= 15;
    flags.push({ severity: "WATCH", message: `Promoter pledging at ${pledge}% — monitor` });
  } else if (pledge > 10) {
    score -= 5;
    flags.push({ severity: "INFO", message: `Promoter pledging at ${pledge}%` });
  }
  
  // --- Promoter Holding Trend ---
  const trend = legalData.shareholding?.trend;
  if (trend && trend.length >= 4) {
    const promoterDecline = trend[0].promoterHolding - trend[3].promoterHolding;
    if (promoterDecline < -3) {
      score -= 10;
      flags.push({ severity: "WATCH", message: `Promoter holding declined ${Math.abs(promoterDecline).toFixed(1)}% over 4 quarters` });
    }
  }
  
  // --- FII Trend (positive signal) ---
  if (trend && trend.length >= 4) {
    const fiiIncrease = (legalData.shareholding.latest.fiiHolding || 0) - (trend[3]?.fiiHolding || 0);
    if (fiiIncrease > 2) {
      score += 5; // Positive signal
      flags.push({ severity: "POSITIVE", message: `FII holding increased ${fiiIncrease.toFixed(1)}% — institutional confidence` });
    }
  }
  
  // --- SEBI/Regulatory Notices ---
  const sebiNotices = legalData.announcements

    score -= 10 * sebiNotices;
    flags.push({ severity: "HIGH", message: `${sebiNotices} SEBI-related notice(s) in 12 months` });
  }
  
  return {
    score: Math.max(0, Math.min(100, score)),
    flags: flags
  };
}
```

## E.4 Sentiment Scorer — Weighted Classification

**File:** `backend/engine/sentimentScorer.js`

```javascript
function computeSentimentScore(classifiedArticles) {
  let weightedSum = 0;
  let weightTotal = 0;
  
  const sentimentMap = { "POSITIVE": 1, "NEUTRAL": 0, "NEGATIVE": -1 };
  
  for (const article of classifiedArticles) {
    // Determine recency weight
    const daysAgo = parseDaysAgo(article.date); // "2 days ago" → 2
    let weight;
    if (daysAgo <= 7)       weight = 3;
    else if (daysAgo <= 15) weight = 2;
    else                    weight = 1;
    
    const sentimentValue = sentimentMap[article.sentiment] || 0;
    weightedSum += sentimentValue * weight;
    weightTotal += weight;
  }
  
  // Normalize to 0-100: weighted average is between -1 and +1
  const normalizedScore = weightTotal > 0 
    ? ((weightedSum / weightTotal) + 1) * 50 
    : 50;
  
  // Compute theme counts
  const themeCounts = {};
  for (const article of classifiedArticles) {
    themeCounts[article.topic] = (themeCounts[article.topic] || 0) + 1;
  }
  const topThemes = Object.entries(themeCounts)
    .sort((a, b) => b[1] - a[1])
    .slice(0, 3)
    .map(([theme, count]) => ({ theme, count }));
  
  // Count sentiments
  const positive = classifiedArticles.filter(a => a.sentiment === "POSITIVE").length;
  const negative = classifiedArticles.filter(a => a.sentiment === "NEGATIVE").length;
  const neutral = classifiedArticles.filter(a => a.sentiment === "NEUTRAL").length;
  
  const flags = [];
  const negativeRatio = negative / classifiedArticles.length;
  if (negativeRatio > 0.5) {
    flags.push({ severity: "HIGH", message: `${Math.round(negativeRatio*100)}% negative news sentiment in last 30 days` });
  } else if (negativeRatio > 0.3) {
    flags.push({ severity: "WATCH", message: `Negative news narrative building (${Math.round(negativeRatio*100)}% negative)` });
  }
  
  return {
    score: Math.round(normalizedScore),
    positive, negative, neutral, total: classifiedArticles.length,
    topThemes,
    flags
  };
}
```

```javascript
function computeCompanyIQ(financialScore, legalResult, sentimentResult) {
```

```javascript
    flags.push({ severity: "CRITICAL", icon: "🚨",
      message: `Current ratio at ${allRatios.currentRatio.toFixed(2)} — cannot meet short-term obligations` });
  }
  
  // === HIGH RISK FLAGS (-15 points each) ===
  
  const pledge = legalData.shareholding?.latest?.pledgedPercent || 0;
  if (pledge > 50 && pledge <= 75) {
    flags.push({ severity: "HIGH", icon: "⚠️",
      message: `Promoter pledging at ${pledge}% — significant risk` });
  }
  
  if (allRatios.debtToEquity > 3.0) {
    flags.push({ severity: "HIGH", icon: "⚠️",
      message: `Debt-to-Equity at ${allRatios.debtToEquity.toFixed(2)} — highly leveraged` });
  }
  
  if (sentimentData.negative / sentimentData.total > 0.5) {
    flags.push({ severity: "HIGH", icon: "⚠️",
      message: `${Math.round(sentimentData.negative / sentimentData.total * 100)}% negative news — heavy negative press` });
  }
  
  // === WATCH FLAGS (-8 points each) ===
  
  if (pledge > 30 && pledge <= 50) {
    flags.push({ severity: "WATCH", icon: "🔶",
      message: `Promoter pledging at ${pledge}% — monitor trend` });
  }
  
  if (allRatios.interestCoverage && allRatios.interestCoverage < 2.0) {
    flags.push({ severity: "WATCH", icon: "🔶",
      message: `Interest coverage at ${allRatios.interestCoverage.toFixed(2)} — debt servicing under pressure` });
  }
  
  if (growthMetrics.revenueCAGR3yr !== null && growthMetrics.revenueCAGR3yr < 0) {
    flags.push({ severity: "WATCH", icon: "🔶",
      message: `Revenue declining — 3-year CAGR is ${growthMetrics.revenueCAGR3yr.toFixed(1)}%` });
  }
  
  // === POSITIVE SIGNALS (+5 points each) ===
  
  if (pledge === 0) {
    flags.push({ severity: "POSITIVE", icon: "✅",
      message: "Zero promoter pledging — clean governance" });
  }
  
  if (profits && profits.length >= 4) {
    let growingCount = 0;
    for (let i = 0; i < profits.length - 1 && i < 3; i++) {
      if (profits[i] > profits[i + 1]) growingCount++;
    }
    if (growingCount >= 3) {
      flags.push({ severity: "POSITIVE", icon: "✅",
        message: "3+ consecutive quarters of profit growth" });
    }
  }
  
  if (allRatios.dividendYield && allRatios.dividendYield > 1) {
    flags.push({ severity: "POSITIVE", icon: "✅",
      message: `Dividend yield at ${allRatios.dividendYield.toFixed(1)}% — income stock` });
  }
  
  return flags;
}
```

## E.8 Orchestrator — Parallel Agent Dispatch

**File:** `backend/orchestrator/orchestrator.js`

```javascript
async function generateReport(companyInput, reportType, onProgress) {
  // Step 1: Resolve company
  const company = resolveCompany(companyInput); // From companies.json
  if (!company) throw new Error(`Company "${companyInput}" not found`);
  
  onProgress?.({ stage: "STARTED", message: `Analyzing ${company.name}...` });
  
  // Step 2: Dispatch agents based on report type
  const agentPromises = [];
  
  // Financial Agent — always runs
  agentPromises.push(
    runFinancialAgent(company, (msg) => onProgress?.({ stage: "FINANCIAL", message: msg }))
  );
  
  // Legal Agent — runs for Quick Scan and above
  if (reportType !== "free_score") {
    agentPromises.push(
      runLegalAgent(company, (msg) => onProgress?.({ stage: "LEGAL", message: msg }))
    );
  }
  
  // Sentiment Agent — runs for Standard Report only
  if (reportType === "standard") {
    agentPromises.push(
      runSentimentAgent(company, (msg) => onProgress?.({ stage: "SENTIMENT", message: msg }))
    );
  }
  
  // Step 3: Execute all agents in parallel
  onProgress?.({ stage: "EXTRACTING", message: "Deploying browser agents..." });
  const results = await Promise.allSettled(agentPromises);
  
  // Step 4: Process results (handle partial failures gracefully)
  const financialData = results[0].status === "fulfilled" ? results[0].value : null;
  const legalData = results[1]?.status === "fulfilled" ? results[1]?.value : null;
  const sentimentData = results[2]?.status === "fulfilled" ? results[2]?.value : null;
  
  if (!financialData) throw new Error("Financial data extraction failed — cannot compute report");
  
  // Step 5: Compute scores
  onProgress?.({ stage: "COMPUTING", message: "Computing financial ratios..." });
  const allRatios = computeAllRatios(financialData);
  const financialScore = computeFinancialScore(allRatios);
  const legalResult = legalData ? computeLegalScore(legalData) : { score: 50, flags: [{ severity: "INFO", message: "Legal data unavailable" }] };
  const sentimentResult = sentimentData ? computeSentimentScore(sentimentData.classified) : { score: 50, flags: [], positive: 0, negative: 0, neutral: 0, total: 0, topThemes: [] };
  
  // Step 6: Composite score + red flags
  const redFlags = detectRedFlags(allRatios, legalData || {}, sentimentResult, allRatios.growth);
  const composite = computeCompanyIQ(financialScore, legalResult, sentimentResult);
  
  // Step 7: LLM Synthesis
  onProgress?.({ stage: "SYNTHESIZING", message: "Generating AI executive summary..." });
  const synthesis = await generateSynthesis(company.name, company.ticker, {
    financial: { ...financialData, ratios: allRatios, score: financialScore },
    legal: { ...legalData, score: legalResult.score },
    sentiment: sentimentResult,
    redFlags,
    compositeScore: composite.score,
    rating: composite.rating
  });
  
  // Step 8: Assemble final report
  onProgress?.({ stage: "COMPLETE", message: "Report ready!" });
  
  return {
    company: company.name,
    ticker: company.ticker,
    sector: company.sector,
    reportType,
    generatedAt: new Date().toISOString(),
    companyIQ: composite,
    financial: { raw: financialData, ratios: allRatios, score: financialScore },
    legal: legalData ? { raw: legalData, score: legalResult.score, flags: legalResult.flags } : null,
    sentiment: sentimentData ? { raw: sentimentData, ...sentimentResult } : null,
    redFlags,
    synthesis
  };
}
```

## F.6 Caching & Fallback Layer

**File:** `backend/cache/cacheLayer.js`

```javascript
const NodeCache = require("node-cache");
const cache = new NodeCache({ stdTTL: 86400 }); // 24 hours

async function getCachedOrFresh(key, freshFn) {
  const cached = cache.get(key);
  if (cached) {
    return { data: cached, source: "cache", cost: 0 };
  }
  
  try {
    const fresh = await freshFn();
    cache.set(key, fresh);
    return { data: fresh, source: "live", cost: "calculated" };
  } catch (error) {
    // Fallback: return stale cache if available, or error
    const stale = cache.get(key); // ttl expired but might still be in memory
    if (stale) return { data: stale, source: "stale_cache", cost: 0 };
    throw error;
  }
}
```

**Pre-cache strategy for demo:**
- Before recording the demo video, run reports for 5 companies to warm the cache
- During demo, if TinyFish is slow, cached data is served instantly
- UI shows "Data updated: 2 hours ago" for cached data
